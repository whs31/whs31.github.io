Подробное описание

Class for working with local application settings, stored in filesystem.

Provides access to reading and writing application settings to local .ini file. Automatically load default settings on fresh application install. Class mainly used as wrapper over QSettings instance, providing useful invokable functions and properties avialable in QML.

Config class also provides useful macro for accessing it's properties from different classes and namespaces:

CONFIG(property_name);

Example usage of macro:

Network::startTelemetrySocket(CONFIG(remoteIP) + ":" + CONFIG(telemetryPort), CONFIG(telemetryFrequency));
Config::Config::remoteIP
QString remoteIP
IPv4-address of SAR (DE10-NANO).
Definition: config.h:47
Config::Config::telemetryFrequency
float telemetryFrequency
Telemetry package interval in seconds.
Definition: config.h:175
Config::Config::telemetryPort
QString telemetryPort
Port for requesting telemetry.
Definition: config.h:109

In QML, exactly same code can be achieved using following example:

Network.startTelemetrySocket(Config.remoteIP + ":" + Config.telemetryPort, Config.telemetryFrequency);

◆ useRadians

useRadians
readwrite

Compatibility setting for older firmware versions.

If boolean evalueates to true, image metadata decoding process will expect radian values in azimuth, drift angle and div fields. VT45-firmware (target platform) currently behaves like that. User needs to manually uncheck this boolean, if resulting images are poor cut and incorrectly aligned. By default, this property returns true.

useRadians(), setUseRadians(), useRadiansChanged()
