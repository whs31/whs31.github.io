Подробное описание

Публичный интерфейс для работы с сетевой составляющей РЛС "Квазар".

Представляет из себя класс-синглтон, содержащий набор функций для взаимодействия с РЛС "Квазар". Большая часть функций и свойств доступна в движке QML.

Класс автоматически регистрирует себя и QuasarSDK::Enums в QML при первом вызове функции get().

Производит попытку подключения к РЛС по указанным адресам.

Функция запускает все дочерние сокеты с выбранными параметрами и адресами.

Аргументы
telemetry_request_address- IP-адрес для запросов телеметрии в формате 192.168.1.0:5555.
telemetry_receive_address- IP-адрес для получения телеметрии.
telemetry_frequency- интервал в секундах между пакетами телеметрии.
execd_address- IP-адрес для команд в сервис execd на РЛС.
feedback_address- IP-адрес для приема потока stdout с РЛС.
tcp_lfs_address- IP-адрес для сервера TCP-IP.
udp_lfs_address- IP-адрес для сокета полосовых изображений.
Заметки
Может быть вызвана из QML через мета-объектную систему.

◆ startPings()

noexcept

Начинает проверку 5-ти выбранных адресов на доступность.

После вызова этой функции будут запущены 5 хостов консоли с командой ping, которые будут проверять на доступность адреса, заданные функцией setPingAddressList.

Заметки
Может быть вызвана из QML через мета-объектную систему.
Предупреждения
Эта функция должна быть вызвана после того, как будут заданы адреса для проверки.

noexcept

Задает список адресов для проверки на доступность.

Порядок адресов следующий:

  • IP-адрес DE10-NANO
  • IP-адрес NVidia Jetson
  • IP-адрес бортовой навигации
  • IP-адрес коммутатора антенн
  • IP-адрес коммутатора питания Адреса должны предоставляться без портов (напр. 127.0.0.1).
    Аргументы
    list- массив из пяти адресов.
    Предупреждения
    Функция должна быть вызвана перед вызовом метода startPings.
